github是一个远端代码仓库托管平台，类似百度云的概念

一、【远端仓库】创建代码创库
	1、注册github帐号
	2、在帐号里创建一个空代码仓库，可以理解为在自己的百度云里创建的一个项目的文件夹（你已经完成这一步了）

二、【克隆\下载】这个空仓库文件夹到自己的电脑
	涉及的命令：git clone + 你远端仓库地址
	例如 ： git clone https://git.neoteched.com/yeh/winterfell.git

	注意说明：远端仓库地址有两种格式（https或SSH）
	https的形式 https://git.neoteched.com/yeh/winterfell.git
	SSH的形式 git@git.neoteched.com:yeh/winterfell.git

	这两种形式都是你对应远端仓库的地址，只是不一样的登录连接形式，https的形式会在你每次要从远端下载代码或提交本地代码到远端的时候输入你的github帐号和密码。HSS的形式就省去输入帐号和密码的形式了，但需要配置本地电脑的ssh-key到你的远端github账户里，类似一个电脑唯一识别码，目的是为了让github自动认出你的身份。


git bash是命令窗口的意思，是命令方式操作git,他可以完成所有git操作

三、关于本地仓库，可以理解为那个.git文件夹就是本地真正的仓库位置，但你不能将代码直接复制进去，你只能将代码放在和他一个目录的test-vue文件夹里

四、【git add .】添加当前目录里的所有文件到本地仓库.git里；
	这个命令只是将文件进入提交状态，但还没有真正提交到.git本地仓库；

五、【git commit -m 添加本次提交的备注说明】备注部分是必填的，这个操作执行完代码就真正的提交到本地.git仓库里了。



到此为止已经完成代码保存到本地仓库的所有操作了，如果代码只是自己用或者没有网或者还不打算保存到远端，在自己一台电脑上开发，每次自己修改代码或添加新代码后，add和commit就可以不断的将自己每天开发的代码备份到本地仓库，每次commit自己做好相关说明注释，就可以方便自己管理自己的代码了，具体如何查看或还原自己add、commit的记录后边在了解就行。

但很多时候还是希望自己的代码可以提交上传保存到远端仓库，这样就可以让别人克隆下载clone，或自己如果换电脑了不用担心代码会丢失一样，就像自己的照片百度云里保存一份，自己电脑里保存一份一样，其实本质上这两个仓库都是完全一样的。

六、【git push】将代码提交上传到远端github帐号对应的远端仓库文件夹里
	注意：对于第一次提交代码，如果没有设置本地git关联的远端仓库地址，git是不知道要上传提交到远端哪个地址的，因此需要添加一个远端仓库关联：
	【git remote add origin + 你的远端仓库地址】
	添加完成之后就可以执行git push命令将本地代码提交到远端了，说明git push后边不加其他参数就是上传到默认仓库地址

	说明：你这个是通过【git clone】直接克隆下来的就不用添加远端关联了，因为git知道自己当初是从哪个远端下载下来的，克隆的时候git会记住远端自动关联记录下来，你所以直接git push就行

	【git push -u origin master】这个命令后边几个参数的意思
	-u 的意思是要指定对应要上传的对应的地址分支
	origin 远端地址的别名
	master 相应分支

	你先学死记住会用日后再一点点了解更多参数的用法。

七、【git pull】下载远端的最新代码到本地仓库，这样别人提交上传到远端的代码也就会一并下载回来。
